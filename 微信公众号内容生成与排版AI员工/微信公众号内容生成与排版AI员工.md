# AI员工需求：微信公众号内容生成与排版AI员工

## 0. 为什么需要这个AI？
- **现状痛点**：每周需要发布3-5篇公众号文章，从选题、撰写、配图到排版，一篇文章需要2-3小时；153编辑器会员费用较高，且需要手动操作
- **参考案例**：类似Jasper AI的内容生成 + Canva的自动排版功能
- **预期价值**：节省70%内容创作时间，降低编辑器会员成本，提升发布频率和内容质量

---

## 1. 一句话定义
> 当运营人员需要发布公众号文章时，这个AI自动根据素材库和内容类型生成图文并茂的公众号文章，并使用开源编辑器自动排版，输出可直接复制到微信公众号后台的HTML格式内容，省去2-3小时的撰写和排版工作。

## 4. 处理流程（伪代码/流程图）

```
1. 接收输入并识别内容类型
   ├─ 验证必填字段（content_type, key_points/topic）
   └─ 根据content_type加载对应模板

2. 素材采集阶段
   ├─ 如果是"新品上市/装机案例/获奖推送/营销案例"
   │   └─ 从企业素材库检索相关内容（公司介绍.md + 产品卖点 + 营销案例）
   ├─ 如果是"节日促销/喝水科普"
   │   ├─ 调用网络爬虫获取节日热点/健康科普素材（小红书/知乎热门）
   │   └─ 结合企业素材库的产品信息
   └─ 从D:\university\Pudow\Assets\产品实拍图 获取对应产品图片

3. 内容生成阶段
   ├─ 调用LLM生成文章框架（标题 + 大纲）
   ├─ 根据大纲逐段生成内容
   │   ├─ 开头：痛点场景 + 引出产品
   │   ├─ 正文：产品卖点展开（结合企业素材库的技术点）
   │   ├─ 案例：引用营销案例（华为/阿里/复星等真实客户）
   │   └─ 结尾：行动召唤（联系方式/阅读原文）
   └─ 生成Markdown格式内容

4. 图片匹配与插入
   ├─ 解析image_requirement
   ├─ 从素材库匹配对应图片
   │   ├─ 产品图：匹配产品名称
   │   ├─ 功能示意图：匹配功能关键词（流量/滤芯/IoT）
   │   └─ 氛围图：如缺失则调用AI生图（DALL-E/Midjourney API）
   └─ 在内容中标记图片插入位置

5. 自动排版阶段
   ├─ 加载开源编辑器模板（模仿秀米风格）
   │   ├─ 标题样式：居中、大字号、品牌色（朴道蓝 #0066CC）
   │   ├─ 正文样式：两端对齐、行间距1.75、字号16px
   │   ├─ 图片样式：居中、圆角、阴影效果
   │   └─ 分割线/引用块/按钮样式
   ├─ 将Markdown转换为HTML
   ├─ 应用CSS样式（内联样式，适配微信公众号）
   └─ 插入图片（转为<img>标签，src指向临时URL或base64）

6. 质量检查
   ├─ 敏感词检测（政治/竞品贬低）
   ├─ 字数检查（建议800-1500字）
   ├─ 图片数量检查（建议3-8张）
   └─ 链接有效性检查

7. 输出与预览
   ├─ 生成JSON格式结果
   ├─ 提供HTML预览链接
   └─ 标注"人工审核要点"（产品参数准确性、案例真实性）
```

## 5. 边界与限制（防呆设计）

### 异常情况处理
- **如果素材库中找不到对应产品图片** → 输出"[缺失：产品名称图片]"占位符，提示人工补充
- **如果网络爬取失败（节日素材/科普内容）** → 降级使用企业素材库通用内容 + 提示"本次未引用外部热点"
- **如果生成内容包含竞品名称（美的/沁园/安吉尔）** → 自动替换为"传统净水设备"或删除
- **如果生成内容过短（<500字）** → 自动扩展"客户案例"或"产品优势对比"段落
- **如果排版后HTML超过微信限制（>30MB）** → 压缩图片质量或转为外链图片

### 安全限制
- 处理超时限制：单篇文章生成 < 60秒
- 图片大小限制：单张图片 < 2MB，总图片数 ≤ 10张
- 敏感词库：加载《广告法禁用词库》+ 《净水行业负面词库》
- 数据隐私：不在文章中泄露客户联系方式、合同金额等商业机密

### 内容规范
- 标题禁止使用"最好/第一/顶级"等绝对化用词
- 产品参数必须与企业素材库一致（如2000G流量不能写成2500G）
- 客户案例必须是企业素材库中已有的真实案例
- 价格信息必须标注"仅供参考，以实际报价为准"

## 6. 技术约束

### 性能要求
- **预期响应时间**：单篇文章生成 < 60秒（初期可放宽到120秒）
- **并发支持**：支持3个内容同时生成（对应3个运营人员）
- **排版兼容性**：生成的HTML必须在微信公众号后台、企业微信、订阅号助手中显示一致

### 依赖资源
- **外部服务**：
  - LLM API：Claude/GPT-4（内容生成）
  - AI绘图API：DALL-E 3 或 Midjourney（备用，优先使用素材库图片）
  - 网络爬虫：Playwright/Selenium（爬取小红书/知乎热点）
  - 图床服务：阿里云OSS/腾讯云COS（图片托管）
- **本地资源**：
  - 企业素材库路径：`D:\university\Pudow\xuqiu\公司相关介绍.md`
  - 产品图片路径：`D:\university\Pudow\Assets\产品实拍图\`
  - 开源编辑器模板库：`\templates\wechat_editor\`（预置20套模板）
- **API权限**：读写企业素材库、上传图片到图床、发送预览邮件
- **成本预估**：
  - 每篇文章LLM成本：¥0.5-1元（约10K tokens）
  - 图片生成成本（如需）：¥0.3元/张
  - 月发布60篇文章，总成本约 ¥50-80元

### 技术栈建议
- 后端：Python FastAPI（异步处理）
- LLM调用：LangChain（便于切换不同模型）
- 网络爬虫：Playwright（支持动态页面）
- 内容转换：python-markdown + BeautifulSoup（Markdown转HTML）
- 图片处理：Pillow（压缩、水印）
- 数据存储：
  - 素材库索引：Elasticsearch（快速检索）
  - 生成历史：SQLite（记录每篇文章生成参数，便于复用）

## 7. 如何判断做对了？（验收标准）

### 定量指标
- 测试集准确率：产品参数准确率 > 95%（与素材库对比）
- 响应时间：90%的文章生成 < 60秒
- 图文匹配度：AI选择的图片与内容相关性人工评分 > 4.0/5.0
- 排版合格率：HTML在微信公众号后台显示正常率 > 98%

### 定性检查
- 用案例1测试，生成的文章必须包含：产品名称、核心卖点、真实客户案例
- 用案例2测试，生成的促销文章必须包含：促销力度、截止日期、购买引导
- 用案例3测试，科普文章必须自然嵌入朴道产品优势，不能生硬植入
- 排版风格符合"专业、简洁、科技感"定位，不花哨

## 附录：实际工作流建议

### 运营人员操作流程
1. **周一上午**：在AI后台选择本周要发布的内容类型（如：2篇新品推广 + 1篇节日促销 + 2篇健康科普）
2. **AI生成**：10分钟内AI生成5篇文章初稿，以列表形式展示标题和配图预览
3. **人工筛选**：运营人员浏览列表，选择3篇质量较好的进入详细编辑
4. **微调审核**：
   - 检查产品参数是否准确（对照素材库）
   - 调整标题（让标题更吸引人）
   - 替换或新增图片（如AI选的图不满意）
   - 修改结尾CTA（根据当前营销重点）
5. **一键复制**：点击"复制HTML"，粘贴到微信公众号后台
6. **发布**：在公众号后台预览无误后定时发布

### 素材库维护建议
- **每月更新**：新增当月装机案例、获奖信息到素材库
- **图片管理**：产品实拍图按产品型号分文件夹存放，命名规范如`名士K2_主图_01.jpg`
- **案例库**：每个客户案例包含：客户名称、行业、痛点、解决方案、效果数据（如节省30%成本）

### 质量保障机制
- **每周抽查**：运营主管抽查本周发布的3篇文章，评分并反馈给AI优化
- **用户反馈**：在AI后台设置"本文生成质量"评分按钮（1-5星），用于训练优化
- **季度复盘**：每季度对比"AI生成文章"vs"人工撰写文章"的阅读数据，调整AI生成策略
